# AGENTS Instructions

This repository hosts the **ActivityPlanner** project. The README contains the full German project description.

## Project summary
- Private planning web app for small friend groups.
- Core features: username+password registration, profile pictures, Tinder-like activity carousel with join/decline buttons, detailed activity views, end-looped sorted carousel, support for spontaneous actions ("Kommt her"), ready status indicator, and extra profile views (rejected items, past activities, calendar).
- Technical requirements: can be hosted locally on Mac or Windows, optional low-cost cloud (<10 €/month), low memory usage, standard resolution images, push/browser notifications, and strong security against CSRF/XSS/SQL injection.
- UX requirements: modern look with smooth animations, consistent color coding (green = positive, red = negative).

## Development guidelines
- Keep the implementation lightweight and focused on browser-based delivery.
- Aim for responsive design.
- Use ChatGPT Codex for code generation during development.

There are currently no tests or code style rules defined.

## ActivityPlanner – Überblick, Funktionen und Installation

ActivityPlanner – Überblick, Funktionen und Installation

Übersicht der App

ActivityPlanner ist eine schlanke Webanwendung, mit der ein kleiner Freundeskreis auf einfache Weise gemeinsame Aktivitäten planen kann. Die App wird lokal auf einem Rechner (z. B. einem MacBook M1 oder Windows-PC) gehostet und nur innerhalb des Freundeskreises genutzt. Ihr Zweck ist es, Events schnell zu organisieren und jederzeit sichtbar zu machen, wer daran teilnimmt. Dabei legt die App Wert auf Übersicht und Schnelligkeit – sie fungiert als fokussierte, private Planungsplattform mit maximaler Übersicht, blitzschnellen Reaktionen und minimalem Overhead, perfekt abgestimmt auf kleine Freundeskreise.

Hauptfunktionen: Die Anwendung bietet Nutzerkonten mit minimalen Anforderungen – ein Account besteht lediglich aus einem Benutzernamen und Passwort (keine E-Mail-Adresse erforderlich) und einem hochgeladenen Profilbild. Über ein zentrales Karten-Karussell im Tinder-Stil können Aktivitäten vorgeschlagen werden, zu denen Nutzer mit „Ich bin dabei“ (Zusagen) oder „Ich bin raus“ (Absagen) reagieren. Durch Antippen einer Aktivitätskarte öffnet sich eine Detailansicht mit ausführlicher Beschreibung des Events sowie einer vollständigen Liste aller Teilnehmer; dort kann ein Nutzer seine Zusage auch nachträglich ändern. Abgelehnte Aktivitäten verschwinden zwar aus der Hauptansicht, bleiben jedoch erhalten und können im Bereich „Abgelehnt“ später wiederhergestellt werden. Zusätzlich gibt es die Möglichkeit, spontane Aktionen über den „Kommt her“-Button anzustoßen – dabei genügt es, einen Titel und eine kurze Beschreibung einzugeben, woraufhin alle Nutzer sofort eine Benachrichtigung erhalten; diese spontane Aktion erscheint dann als hervorgehobene erste Karte im Karussell. Über einen „Ready“-Status unter dem Profilbild können Nutzer signalisieren, dass sie aktuell bereit für spontane Aktivitäten sind; aktivierte Nutzer erscheinen in einer schmalen Leiste oben rechts neben dem „Kommt her“-Button. Weitere Ansichten sind im Profilmenü zugänglich, darunter „Vergangene Aktivitäten“ (eine chronologische Liste erledigter Events) und ein Kalender-Tab mit allen angenommenen Aktivitäten in Datum/Uhrzeit-Übersicht.

Technische Basis: Die App ist mit Node.js (CommonJS-Modulformat) und dem Express-Framework umgesetzt. Sie verwendet eine leichtgewichtige lokale Datenbank (SQLite über better-sqlite3) bzw. JSON-Dateien zur Datenspeicherung, z. B. für Nutzer und Aktivitäten. Für sicherheitsrelevante Funktionen kommen u. a. helmet, csurf (CSRF-Schutz), validator und sanitize-html zum Einsatz, wodurch ein hoher Schutz vor gängigen Web-Angriffen (CSRF, XSS, SQL-Injection etc.) erreicht wird. Bilder (etwa Profilbilder oder Aktivitätsfotos) werden beim Hochladen mit Hilfe von sharp verarbeitet und in angemessener Auflösung gespeichert, um den Performance-Anforderungen (kleiner Speicherbedarf, keine übertrieben hochauflösenden Medien) zu genügen. Die App unterstützt Push-Benachrichtigungen über die Web Push API: Beim ersten Start werden automatisch VAPID-Schlüssel generiert und an die Clients verteilt; die Nutzer können Browser-Benachrichtigungen abonnieren. So erhalten z. B. die Ersteller von Aktivitäten eine Push-Mitteilung, wenn jemand „Ich bin dabei“ klickt, und bei „Kommt her“-Aktionen werden sofort alle Mitglieder des Freundeskreises benachrichtigt. Im Hintergrund läuft ein WebSocket-Server (ws), der für Echtzeit-Updates sorgt (z. B. Aktualisierung des Teilnehmerstatus in allen offenen Browserfenstern). Die Anwendung ist plattformunabhängig in jedem modernen Webbrowser nutzbar und wurde auf ein responsives Design ausgelegt. Sie kann lokal auf gängigen Systemen (macOS, Windows) betrieben oder optional kostengünstig in der Cloud gehostet werden (vorgesehene Budgetgrenze ≤ 10 € pro Monat).

Beschreibung der Benutzeroberfläche

Design und Layout: Die Oberfläche von ActivityPlanner ist modern gestaltet (geschmeidige Animationen, zeitgemäßes Layout) und auf Klarheit optimiert. Wichtige Aktionen sind farblich kodiert: Grün signalisiert positive Aktionen (Teilnahme) und Rot negative Aktionen (Ablehnung). Die App bietet sogar eine Umschaltmöglichkeit für den Dark Mode (dunkles/helles Farbschema) über einen kleinen Mond-Button in der Navigationsleiste, um auch bei schlechten Lichtverhältnissen eine angenehme Nutzung zu ermöglichen.

Hauptansicht (Home): Nach dem Einloggen gelangen die Nutzer zur Hauptansicht, die zentrales Karten-Karussell und eine Toolbar am oberen Bildschirmrand enthält. Im Karussell wird jede geplante Aktivität als Karte dargestellt, mit folgendem Inhalt: Titel der Aktivität (oben), Datum bzw. Uhrzeit (in kleinerer Schrift darunter), ein passendes Bild in der Kartenmitte (vom Ersteller beim Anlegen ausgewählt) und am Kartenende eine Leiste mit runden Profil-Icons aller Nutzer, die zugesagt haben. Unter jeder Karte befinden sich zwei auffällige Buttons: „Ich bin dabei“ (grün hinterlegt) und „Ich bin raus“ (rot hinterlegt). Damit können die Nutzer schnell ihre Teilnahme zu- oder absagen, ähnlich dem Prinzip einer Tinder-Karte. Das Karussell ist als Endlosschleife umgesetzt – nach der letzten Karte folgt wieder die erste. Standardmäßig sind die Karten chronologisch nach Datum sortiert, so dass das nächste anstehende Event vorne steht. Sobald ein Nutzer eine Aktivität mit „Ich bin raus“ ablehnt, verschwindet diese Karte umgehend aus seinem Karussell. Sie ist jedoch nicht gelöscht, sondern unter dem Profilmenüpunkt „Abgelehnt“ jederzeit einsehbar und kann von dort aus bei Bedarf reaktiviert werden.

Toolbar und Navigation: Die obere Leiste enthält links den „Kommt her“-Button (grün), über den spontane Aktionen gestartet werden können. Ein Klick darauf öffnet ein kleines Dialogfenster, in dem lediglich ein Titel und eine Beschreibung einzugeben sind. Bestätigt der Nutzer mit Senden, erhalten alle Mitglieder des Freundeskreises sofort eine Push-Benachrichtigung, und im Karussell der Hauptansicht erscheint eine neue Aktivitätskarte mit auffälligem Rahmen ganz vorne. Rechts in der Toolbar befindet sich das Profil-Menü des eingeloggten Nutzers: Es zeigt als Icon das kleine runde Profilbild des Nutzers. Ein Klick darauf öffnet ein Dropdown-Menü mit folgenden Einträgen: Abgelehnt (für abgelehnte Aktivitäten), Vergangene Aktivitäten (Liste aller bereits durchgeführten Events in chronologischer Reihenfolge) und Kalender (tabellarische Übersicht aller zugesagten zukünftigen Termine). Über diesen Menüzugriff können Nutzer also auch ältere oder abgelehnte Events einsehen sowie alle kommenden Teilnahmen in einem Kalenderformat betrachten. Unter dem Profilbild in der Toolbar befindet sich zudem der „Ready“-Schalter: ein Button, mit dem der Nutzer signalisieren kann, dass er aktuell bereit ist für spontane Treffen. Sobald aktiviert, erscheint ein kleiner Hinweis in der Toolbar (die Ready-Leiste oben rechts) und der eigene Name/Avatar taucht dort für andere sichtbar als bereit auf. So können Freunde auf einen Blick sehen, wer gerade Zeit und Lust auf eine spontane Aktion hätte. Daneben erlaubt ein weiterer kleiner Button (Mond-Symbol) das Umschalten zwischen Hell- und Dunkelmodus der App-Oberfläche, um je nach Vorliebe oder Tageszeit die optimale Darstellung zu wählen.

Detailansicht: Klickt ein Nutzer auf das Bild einer Aktivitätskarte, öffnet sich modal die Detailansicht zu dieser Aktivität. In dieser Ansicht werden der vollständige Titel und die Beschreibung der Aktivität angezeigt – die Beschreibung kann dabei auch Links enthalten (diese werden vom System erlaubt und entsprechend sicher angezeigt). Darunter listet die Detailansicht alle Teilnehmer mit Namen auf, die der Aktivität zugesagt haben. Am unteren Rand der Detailansicht hat der Nutzer außerdem die Möglichkeit, seinen eigenen Zusagestatus zu ändern (z. B. doch noch absagen oder zusagen). Ein Klick auf den „Status ändern“-Button schließt die Detailansicht und aktualisiert entsprechend die Anzeige im Karussell.

Installationsanleitung

Im Folgenden wird die Einrichtung der App Schritt für Schritt erläutert. Systemvoraussetzungen: Sie benötigen Node.js in Version 18 oder höher (die App wurde auf Node.js v18 entwickelt) sowie einen Webbrowser. Unterstützt werden lokale Installationen unter macOS (auch Apple M1) und Windows. Für optionale Schritte sollten Sie Git (für den Repository-Klon) und Docker installiert haben. Tipp: Für die Entwicklung bzw. Betrachtung des Quellcodes ist ein Code-Editor wie Visual Studio Code empfehlenswert.

Schritt-für-Schritt Installation (lokal)
1.Visual Studio Code installieren (optional): Laden Sie bei Bedarf VS Code von der offiziellen Website herunter und installieren Sie es für Ihr Betriebssystem. Dieser Schritt ist zum Ausführen der App nicht zwingend nötig, erleichtert aber Entwicklung und Konfiguration.
2.Quellcode beziehen: Besorgen Sie sich den Quellcode der Anwendung. Entweder klonen Sie das Git-Repository via Terminal (git clone <repo-url>) oder Sie laden das ZIP-Archiv herunter und entpacken es in ein Verzeichnis Ihrer Wahl. (Falls Sie dieses Dokument lesen, haben Sie vermutlich bereits das Archiv ActivityPlanner-main.zip erhalten – entpacken Sie es entsprechend.)
3.Projektordner öffnen: Öffnen Sie den entpackten Projektordner in VS Code oder einem Editor Ihrer Wahl, um ggf. Dateien bearbeiten zu können. In VS Code wählen Sie dazu z. B. Datei → Ordner öffnen… und navigieren zum Projektverzeichnis.
4.Node.js installieren: Stellen Sie sicher, dass Node.js in Version 18 oder höher auf Ihrem System installiert ist. Die aktuelle Version erhalten Sie auf nodejs.org. Überspringen Sie diesen Schritt, falls Node bereits vorhanden ist.
5.Abhängigkeiten installieren: Öffnen Sie ein Terminal/Command Prompt im Projektordner (dort, wo sich die Datei package.json befindet). Führen Sie den Befehl npm install aus, um alle erforderlichen Pakete zu installieren. Dieser Schritt lädt alle im Projekt benötigten Bibliotheken (wie Express, SQLite usw.) herunter.
6.App starten: Starten Sie den Entwicklungsserver mit dem Befehl npm run dev. Die Anwendung wird nun lokal gestartet (standardmäßig auf Port 3000). Im Terminal sollten keine Fehler erscheinen; sobald “Listening on port 3000” o. ä. ausgegeben wird, öffnen Sie einen Webbrowser und rufen die URL http://localhost:3000 auf. Sie sollten die ActivityPlanner-Oberfläche angezeigt bekommen und können die App nun verwenden.
7.Tests ausführen (optional): Das Projekt enthält automatische Tests (mittels Jest und Cypress). Führen Sie npm test im Projektordner aus, um alle Tests laufen zu lassen. Die Testergebnisse erscheinen in der Konsole. Dieser Schritt ist nicht notwendig zum Betrieb der App, gibt aber Auskunft darüber, ob alle Funktionen erwartungsgemäß arbeiten.
8.Docker verwenden (optional): Alternativ zum lokalen Node-Betrieb können Sie die Anwendung in einem Docker-Container ausführen. Voraussetzung ist eine funktionierende Docker-Installation. Nutzen Sie dann im Projektordner den Befehl docker-compose up, welcher den bereitliegenden Docker-Container baut und die App darin startet. Der Dienst steht anschließend ebenfalls auf Port 3000 zur Verfügung. (Die Datei Dockerfile definiert ein geeignetes Node-18-Containerimage, und docker-compose.yml sorgt für dessen Ausführung.) Um die Container wieder zu beenden, drücken Sie im Terminal Strg+C oder führen docker-compose down aus.
9.Cloud-Deployment (optional): Die App kann auch auf Cloud-Plattformen wie fly.io oder Railway bereitgestellt werden, die Node.js-Container unterstützen und in kostenlosen Basisplänen weniger als 10 € pro Monat kosten. Für fly.io genügen wenige Schritte: Nach Installation der Fly-CLI können Sie im Projektordner fly launch ausführen (für die Initialisierung) und anschließend fly deploy, um die App in die Fly-Cloud zu laden. Für Railway verwenden Sie die Railway-CLI: Installieren Sie sie mit npm i -g railway, melden Sie sich an (railway login), initialisieren Sie das Projekt im Ordner (railway init) und deployen Sie es mit railway up. Nach erfolgreicher Bereitstellung gibt die Railway-Konsole die URL aus, unter der die laufende Anwendung erreichbar ist. (Hinweis: Cloud-Deployment ist optional und erfordert entsprechende Konten bei den Anbietern; für reine lokale Nutzung kann dieser Schritt übersprungen werden.)

Hinweise zur Nutzung der App
•Erster Start – Registrierung: Beim erstmaligen Aufruf der App (http://localhost:3000) ist noch kein Benutzerkonto vorhanden. Legen Sie daher zunächst einen Account an. Die Registrierung erfordert lediglich die Eingabe eines gewünschten Benutzernamens und Passworts; eine E-Mail-Adresse wird nicht benötigt. Zudem werden Sie aufgefordert, ein Profilbild hochzuladen, das später überall als rundes Icon neben Ihrem Namen angezeigt wird. (Tipp: Verwenden Sie ein quadratisches Bild für optimale Darstellung.) Nach erfolgreicher Registrierung loggt Sie die App automatisch ein und zeigt die Hauptansicht.
•Aktivitäten-Karussell bedienen: In der Hauptansicht sehen Sie die geplanten Aktivitäten als einzelne Karten im Karussell. Navigieren Sie durch die Karten, um alle vorgeschlagenen Events zu betrachten. Für jedes Event können Sie mittels der beiden Buttons unter der Karte Ihre Teilnahme entscheiden: Klick auf „Ich bin dabei“ markiert Sie als Teilnehmer des Events (grüner Button), Klick auf „Ich bin raus“ lehnt Ihre Teilnahme ab (roter Button). Sie können Ihre Entscheidung später jederzeit revidieren: Öffnen Sie dazu die Detailansicht der Aktivität (siehe unten) und klicken Sie auf Status ändern. Beachten Sie, dass abgelehnte Aktivitäten sofort aus Ihrem Karussell verschwinden, um die Übersicht schlank zu halten. Diese Events sind jedoch nicht gelöscht – Sie finden sie im Profilmenü unter „Abgelehnt“ wieder. Dort können Sie bei Bedarf eine zuvor abgelehnte Aktivität wieder sichtbar machen.
•Detailansicht aufrufen: Ein Klick auf das Bild oder den Titel einer Aktivitätskarte öffnet die Detailansicht für dieses Event. Hier sehen Sie den vollständigen Titel der Aktivität und die ausführliche Beschreibung. Die Beschreibung kann auch Weblinks enthalten – diese sind erlaubt und können z. B. für weiterführende Informationen genutzt werden. Darunter listet die Detailansicht alle Teilnehmer namentlich auf, die aktuell zugesagt haben. Unten befindet sich der Button „Status ändern“, mit dem Sie Ihre eigene Zusage/Ablehnung für dieses Event bearbeiten können. Schließen können Sie die Detailansicht durch einen Klick außerhalb des Modalfensters oder auf den Hintergrund.
•Neue Aktivität hinzufügen: Um selbst eine Aktivität vorzuschlagen, stehen Ihnen zwei Wege zur Verfügung. Geplante Events mit definiertem Datum und Bild können über das Backend hinzugefügt werden (z. B. mittels eines Formulars, falls implementiert, oder direkt in der Datenbasis). Für spontane Treffen hingegen nutzen Sie den „Kommt her“-Button oben links. Dieser öffnet ein einfaches Formular, in das Sie lediglich einen Titel und eine Beschreibung der spontanen Aktion eingeben. Nach dem Absenden (Senden-Button) wird sofort eine Benachrichtigung an alle Nutzer verschickt, und das Event erscheint als hervorgehobene Karte im Karussell aller Nutzer. Verwenden Sie “Kommt her” beispielsweise, um spontan alle auf einen Kaffee einzuladen – jeder sieht die Benachrichtigung und kann direkt in der App zusagen oder ablehnen.
•„Ready“-Status nutzen: Unter Ihrem Profilbild finden Sie den Ready-Schalter. Wenn Sie gerade Zeit und Lust auf spontane Aktivitäten haben, können Sie diesen Button aktivieren. Daraufhin werden Sie für alle Freunde als „bereit“ kenntlich gemacht: In der oberen Leiste erscheint neben „Kommt her“ eine Anzeige (Ready-Leiste), in der Ihr Name/Avatar unter den aktuell bereiten Nutzern auftaucht. Dies signalisiert allen: Sie stehen kurzfristig für Unternehmungen zur Verfügung. Sie können den Ready-Status jederzeit durch erneutes Klicken deaktivieren. Die Ready-Liste ist insbesondere hilfreich, um spontan verfügbare Leute zu finden, ohne jedes Mal nachfragen zu müssen.
•Profilmenü und weitere Seiten: Klicken Sie auf Ihr Profilbild in der Toolbar, um das Profilmenü zu öffnen. Dort gelangen Sie zu weiteren Ansichten:
•Abgelehnt: Hier werden alle Aktivitäten aufgeführt, die Sie mit „Ich bin raus“ abgelehnt haben. Sie können diese Liste nutzen, um eventuell doch noch bei einem früher abgelehnten Event mitzumachen – wählen Sie das Event aus, um es ins Karussell zurückzuholen.
•Vergangene Aktivitäten: In dieser Ansicht sind alle bereits durchgeführten (abgeschlossenen) Events chronologisch aufgelistet. So können Sie vergangene Treffen nachvollziehen oder Details später noch einmal nachlesen.
•Kalender: Diese Seite bietet eine tabellarische Kalenderübersicht aller zukünftigen Aktivitäten, zu denen Sie zugesagt haben. Die Einträge sind nach Datum und Uhrzeit sortiert und geben Ihnen einen schnellen Überblick über anstehende Termine in Ihrem Freundeskreis.
•Benachrichtigungen: ActivityPlanner unterstützt Browser-Push-Notifications für wichtige Ereignisse. Beim ersten Login fragt Sie die Anwendung, ob Sie Benachrichtigungen erlauben möchten (stellen Sie sicher, dass Ihr Browser diese Anfrage nicht blockiert). Wenn Sie zustimmen, erhalten Sie automatische Push-Benachrichtigungen zu folgenden Anlässen:
•Sobald ein Nutzer auf „Kommt her“ eine spontane Aktion erstellt, wird an alle anderen eine Sofort-Benachrichtigung gesendet, damit jeder unmittelbar Bescheid weiß.
•Wenn Sie selbst eine Aktivität erstellt haben (sei es spontan via “Kommt her” oder anderweitig geplant) und andere Nutzer klicken „Ich bin dabei“, erhalten Sie eine Benachrichtigung, dass die Person zugesagt hat. So bleiben Sie immer auf dem Laufenden, wer zu Ihrem Event erscheint.
•Zusätzlich kann die App Benachrichtigungen verwenden, um Erwähnungen oder direkte Ansprachen (z. B. via speziellen „Ping“-Funktionen im Code) an den betreffenden Nutzer zu senden.
•(Hinweis: Die Push-Funktion wird durch einen Service Worker im Browser ermöglicht; stellen Sie sicher, dass Sie beim Laden der Seite das Registrieren dieses Service Workers zulassen.)
•Sicherheit und Sitzung: Da die Anwendung primär für private Nutzung gedacht ist, erfolgen Logins ohne E-Mail-Bestätigung. Alle Daten bleiben lokal auf dem Host gespeichert (sofern Sie die App nicht in der Cloud betreiben). Die Session eines eingeloggten Nutzers ist aus Sicherheitsgründen zeitlich begrenzt (Standard: automatische Abmeldung nach ca. 10 Minuten Inaktivität), Sie können sich danach aber jederzeit erneut einloggen. Es wird empfohlen, für den Produktivbetrieb einen eigenen geheimen Session-Schlüssel zu setzen (Standard ist in der Datei index.js als "change_this_secret" angegeben) und gegebenenfalls HTTPS zu verwenden, wenn die App über ein Netzwerk geteilt wird.

Mit dieser Anleitung und den Hinweisen sollten Sie in der Lage sein, ActivityPlanner erfolgreich zu installieren, zu starten und im Freundeskreis produktiv einzusetzen. Viel Spaß beim Planen Ihrer gemeinsamen Aktivitäten!
